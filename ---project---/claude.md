# このプロジェクトが満たすべき課題要項

• 新しいコマンドを待機しているときにプロンプトを表示する。
• 動作履歴を持つ。

• 適切な実行ファイルを検索して起動する

（PATH 変数に基づいて、または相対パスまたは絶対パスを使用して）。

- シグナル
• 受信したシグナルを示すグローバル変数は最大で1つとする。

    このアプローチの影響を考慮すると、シグナルハンドラがメインのデータ構造にアクセスしなくなることが保証されます。
    注意が必要です。このグローバル変数はシグナル番号のみを格納する必要があり、
    追加情報やデータへのアクセスを提供してはなりません。
    したがって、グローバルスコープで「norm」型の構造体を使用することは
    禁止されています。


閉じられていない引用符や、\（バックスラッシュ）や;（セミコロン）など、対象に必要のない特殊文字は解釈しません。

• '（シングルクォート）を処理します。これにより、シェルは引用符で囲まれたシーケンス内のメタ文字を解釈しません。
• "（二重引用符）を処理します。これにより、シェルは $（ドル記号）を除く引用符で囲まれたシーケンス内のメタ文字を解釈できなくなります。

• 以下のリダイレクトを実装します。
◦ < は入力をリダイレクトします。
◦ > は出力をリダイレクトします。

◦ << には区切り文字を指定し、その区切り文字を含む行が見つかるまで入力を読み取ります。ただし、履歴を更新する必要はありません。

◦ >> は追加モードで出力をリダイレクトします。

• パイプ（| 文字）を実装します。パイプライン内の各コマンドの出力は、パイプを介して次のコマンドの入力に接続されます。

• 環境変数（$ に続く文字列）を処理します。これらは、その値に展開されます。
• $? を処理します。これは、最後に実行されたフォアグラウンドパイプラインの終了ステータスに展開されます。

• bash のように動作する ctrl-C、ctrl-D、および ctrl-\ を処理します。
• 対話モードの場合：
◦ ctrl-C は、新しい行に新しいプロンプトを表示します。
◦ ctrl-Dシェルを終了します。
◦ ctrl-\ は何も行いません。

• シェルには以下の組み込みコマンドが実装されている必要があります。
◦ echo （オプション -n 付き）
◦ cd （相対パスまたは絶対パスのみ）
◦ pwd （オプションなし）
◦ export （オプションなし）
◦ unset （オプションなし）
◦ env （オプションまたは引数なし

◦ exit （オプションなし）

readline() 関数はメモリリークを引き起こす可能性がありますが、修正する必要はありません。
ただし、これはあなた自身のコード、つまりあなたが書いたコードにメモリリークが発生する可能性があるという意味ではありません。

主題の説明に限定してください。求められていないものは
必須ではありません。
要件について疑問がある場合は、bash を参考にしてください。

- Bonus

プログラムは以下の要件を実装する必要があります。
• 優先順位を示す括弧付きの && と ||。
• ワイルドカード * は、現在の作業ディレクトリで使用できます。
