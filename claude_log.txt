  ✅ 正常動作

  - echo, echo -n
  - pwd, env, export, unset
  - 環境変数展開 $VAR
  - シングルクォート（展開抑制）、ダブルクォート（$のみ展開）
  - >, >>, < リダイレクト
  - << EOF heredoc
  - パイプ（複数段も正常）
  - $? コマンド後の更新（true→0, false→1, 未発見→127）
  - 絶対パス・相対パスでの実行
  - cd (引数なし→HOME、存在しないパス→エラー)
  - セミコロンは解釈しない（仕様通り）
  - 未閉じクォートのエラー検出
  - 空行の無視
  - exit N の終了コード伝播（正常）
  - exit 1 2 の「too many arguments」エラー
  - PATH未設定時のコマンド不明エラー
  - ワイルドカード *（カレントディレクトリのみ）

  ❌ 問題あり

  番号: 1
  内容: cd - が動作しない
  詳細: OLDPWD は正しく設定されているが、cd.c に - を OLDPWD
    として解釈する処理がない。chdir("-") を試みて失敗する
  ────────────────────────────────────────
  番号: 2
  内容: $? の初期値が空
  詳細: 起動直後 echo $? が 0 ではなく空文字を返す。env_init() で ?=0
    を設定していないため
  ────────────────────────────────────────
  番号: 3
  内容: exit（引数なし）が常に0で終了
  詳細: exit.c の if (!args[1]) exit(0); が最後のステータスを無視。false 後に exit
    しても終了コードは0
  ────────────────────────────────────────
  番号: 4
  内容: exit abc の終了コードが 255
  詳細: bash では 2 を返すが、minishell は 255 を返す（exit(255) ハードコード）
  ────────────────────────────────────────
  番号: 5
  内容: ワイルドカードがパス付きで展開されない
  詳細: echo src/*.c が展開されず src/*.c リテラルのまま（ボーナスの部分実装）
