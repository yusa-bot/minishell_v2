  4. ビルトイン — 部分的に問題あり

  echo — BUG

  - echo -nnn hello → -nnn hello と出力（bash は hello のみ）
  - bash は - の後に n のみが続く引数をすべて -n オプションとして扱う
  - 修正箇所: src/builtins/echo.c:22 — -n の完全一致でなく、-n+ パターンを受け入れるべき

  5. クォート処理 — 重大なBUG

  src/expander/expand_dollers_quotes.c:36-43 の expand_string にロジックエラー:

  if (process_expand_char(str, &i, quotes, has_wildcard) == -1)
  {
      res = handle_dollar(res, str, &i, env_list);
      ...
  }
  else
      res = append_char(res, str[i++]);  // ← ここが問題

  process_expand_char がクォート文字を処理して 1 を返すとき、関数内で既に i
  をインクリメント済み。しかし else ブランチは返り値 0 と 1
  の両方で実行されるため、クォートの直後の1文字が常にリテラルとして追加される。

  再現例:

  ┌──────────────────┬─────────┬────────────┐
  │       入力       │ 期待値  │ 実際の出力 │
  ├──────────────────┼─────────┼────────────┤
  │ echo "$USER"     │ ayusa   │ $USER      │
  ├──────────────────┼─────────┼────────────┤
  │ echo "$TEST2"    │ hello   │ $TEST2     │
  ├──────────────────┼─────────┼────────────┤
  │ echo '"'$USER'"' │ "ayusa" │ "$USER"    │
  └──────────────────┴─────────┴────────────┘

  修正方法: 返り値 1 のとき continue でスキップ:
  int ret = process_expand_char(str, &i, quotes, has_wildcard);
  if (ret == -1)
      res = handle_dollar(res, str, &i, env_list);
  else if (ret == 0)
      res = append_char(res, str[i++]);
  // ret == 1 の場合は何もしない（i は既にインクリメント済み）
