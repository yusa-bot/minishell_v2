前提条件・全体ルールの確認
メモリリークの不在: ヒープに割り当てられたメモリが終了時に適切に解放されていること（valgrindやleaks等で確認）。

クラッシュの不在: 予期せぬ終了やセグメンテーション違反（Segfault）が発生しないこと。

グローバル変数: 使用している場合、その数と理由が論理的かつ妥当であること。

Mandatory Part（必須要件）
1. コンパイル
make -n を実行し、-Wall -Wextra -Werror フラグが使用されているか。

エラーなしでコンパイルできるか。

Makefileが再リンク（re-link）を行わないか。

2. コマンド実行（パス解決）
絶対パス: /bin/ls のような絶対パスでコマンド（引数なし/あり）が実行できるか。

相対パス: 複雑な相対パス（例: ../../bin/ls）で実行できるか。

環境変数（$PATH）: $PATH を用いてコマンド（ls, wc, awk 等）が実行できるか。

$PATH を空（unset）にした場合、コマンドの実行が失敗するか。

複数のディレクトリを指定した $PATH で、左から右へ順に検索されるか。

3. 組み込みコマンド（Built-ins）
echo: 引数なし、引数あり、-n オプションが正常に動作するか。

cd: ディレクトリの移動（/bin/ls で確認）。相対パスや .、.. が機能するか。

pwd: 複数のディレクトリで正しくカレントパスを出力するか。

env: 現在の環境変数を一覧表示できるか。

export / unset: 環境変数の追加、上書き、削除ができ、env で反映が確認できるか。

exit: 引数あり・なしで適切にシェルを終了するか。

4. クォート処理
シングルクォート ('): 内部の文字列（空白、パイプ、リダイレクト、$USER などの変数）が一切解釈されず、そのまま文字列として扱われるか。

ダブルクォート ("): 内部の空白は保持されつつ、環境変数（$）のみが正しく展開（展開）されるか。

5. 環境変数とステータス
$USER 等の環境変数が正しく展開されるか。

$?（終了ステータス）: 直前のコマンドの終了ステータスを echo $? で正確に取得できるか。正常終了、失敗したコマンド、計算式（expr等）の後のステータスを確認。

6. リダイレクトとパイプ
リダイレクト: <, >, >> が正常に動作するか。失敗するリダイレクトのハンドリング。

ヒアドキュメント (<<): 正常に動作し、かつシェルの履歴（history）を更新しないか。

パイプ (|): 複数コマンド間のパイプ接続が機能するか。途中に存在しないコマンドを挟んだり、リダイレクトと混在させても破綻しないか。

7. シグナルと履歴
Ctrl-C: プロンプトが空の時、入力途中、ブロッキングコマンド（引数なしの cat 等）の実行中のそれぞれで適切に改行され、新しいプロンプトが表示されるか。

Ctrl-\: プロンプトでは何もしないか。ブロッキングコマンド中は適切に終了（Quit）させるか。

Ctrl-D: プロンプトが空の時にシェルを終了するか。

履歴（History）: 上下キーで履歴をナビゲートし、再実行できるか。

ストレステスト: 無効なコマンド（dsbksdgb...）や、大量の引数を渡してもシェルがクラッシュしないか。

Bonus Part（ボーナス要件）
※ 必須要件（Mandatory）が完璧に実装されている場合のみ評価されます。

論理演算子と括弧: &&, ||, およびコマンドのグループ化 () が Bash と同様に動作するか。

ワイルドカード: ローカルディレクトリにおける * が引数として正しく展開されるか。

Surprise (クォートのネスト):

echo "'$USER'" が '実際のユーザー名' を出力するか。

echo '"$USER"' が "$USER" という文字列そのものを出力するか。
